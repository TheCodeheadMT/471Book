## L28: Bayes, Laplace Smoothing

**1. Calculating the Prior**
An intelligence database contains 5,000 intercepted text messages. Analysts have manually labeled 4,500 of them as "Routine" and 500 as "Actionable". What are the prior probabilities $P(\text{Routine})$ and $P(\text{Actionable})$?
<details>
  <summary>Show solution</summary>
  <ul>
  <li><strong>Step 1:</strong> Divide the class count by the total count.<br>
  <i>P</i>(Routine) = 4500 / 5000 = 0.90<br>
  <i>P</i>(Actionable) = 500 / 5000 = 0.10</li>
  <li><strong>Answer:</strong> The priors are <strong>90%</strong> for Routine and <strong>10%</strong> for Actionable.</li>
</ul>
</details>

**2. Calculating the Likelihood**
Within the 500 "Actionable" messages from the previous problem, there is a total word count of 10,000 words. The specific word "coordinates" appears 400 times in these Actionable messages. What is the likelihood $P(\text{"coordinates"} | \text{Actionable})$ without smoothing?

<details>
  <summary>Show solution</summary>
<ul>
  <li><strong>Step 1:</strong> Divide the specific word count by the total class word count.<br>
  <i>P</i>("coordinates" | Actionable) = 400 / 10000 = 0.04</li>
  <li><strong>Answer:</strong> The likelihood is <strong>0.04</strong> (or 4%).</li>
</ul>
</details>

**3. Proportional Posterior Calculation**
We want to classify a new two-word message: "secure location".
Given the following historical data:

* Priors: $P(\text{Urgent}) = 0.2$, $P(\text{Routine}) = 0.8$
* Likelihoods for Urgent: $P(\text{"secure"}|U) = 0.1$, $P(\text{"location"}|U) = 0.3$
* Likelihoods for Routine: $P(\text{"secure"}|R) = 0.05$, $P(\text{"location"}|R) = 0.02$

<br>

Calculate the proportional Naive Bayes score for both classes. Which class does the model predict?

<details>
  <summary>Show solution</summary>
    
<ul>
  <li><strong>Step 1:</strong> Calculate Urgent Score: Prior &times; Likelihoods.<br>
  Score<sub><i>U</i></sub> = 0.2 &times; 0.1 &times; 0.3 = 0.006</li>
  <li><strong>Step 2:</strong> Calculate Routine Score: Prior &times; Likelihoods.<br>
  Score<sub><i>R</i></sub> = 0.8 &times; 0.05 &times; 0.02 = 0.0008</li>
  <li><strong>Answer:</strong> The score for Urgent (0.006) is higher than Routine (0.0008), so the model predicts <strong>Urgent</strong>.</li>
</ul>

</details>

**4. Laplace Smoothing (Unseen Word)**
An adversary sends a message containing the new slang term "ghost". This word has never appeared in your training data for the "Threat" class. The "Threat" class has a total of 2,000 words historically. Your total system vocabulary (all unique words known across all classes combined) is 5,000 words. Using Laplace smoothing ($\alpha = 1$), calculate $P(\text{"ghost"} | \text{Threat})$.

<details>
  <summary>Show solution</summary>
    
<ul>
  <li><strong>Step 1:</strong> Identify variables: Count = 0, Class Total = 2000, |<i>V</i>| = 5000, &alpha; = 1.</li>
  <li><strong>Step 2:</strong> Apply the smoothing formula: <i>P</i> = (Count + &alpha;) / (Class Total + (&alpha; &times; |<i>V</i>|)).<br>
  <i>P</i> = (0 + 1) / (2000 + (1 &times; 5000))</li>
  <li><strong>Step 3:</strong> Solve the fraction.<br>
  <i>P</i> = 1 / 7000 &approx; 0.00014</li>
  <li><strong>Answer:</strong> The smoothed probability is exactly <strong>1/7000</strong> (approx. 0.014%). The math is saved from multiplying by zero.</li>
</ul>

</details>

**5. Laplace Smoothing (Seen Word)**
Using the exact same parameters from Problem 4 (Class Total $= 2000$, $|V| = 5000$, $\alpha = 1$), calculate the smoothed probability for the word "weapon", which historically appeared 50 times in the "Threat" class. Compare it to the un-smoothed probability.

<details>
  <summary>Show solution</summary>
    <ul>
  <li><strong>Step 1:</strong> Calculate un-smoothed probability.<br>
  <i>P</i><sub>un-smoothed</sub> = 50 / 2000 = 0.025</li>
  <li><strong>Step 2:</strong> Calculate smoothed probability.<br>
  <i>P</i><sub>smoothed</sub> = (50 + 1) / (2000 + 5000) = 51 / 7000 &approx; 0.0073</li>
  <li><strong>Answer:</strong> The smoothed probability is <strong>~0.0073</strong>. Notice how smoothing "steals" some probability mass from frequent words to give it to unseen words, lowering the probability of "weapon" from 2.5% to 0.73%.</li>
</ul>
</details>

**6. Mathematical Underflow and Log Probabilities**
A Naive Bayes model processes a 20-word email. Because probabilities are small fractions, multiplying 20 fractions together results in a number so tiny that a computer's memory rounds it down to literal zero (Mathematical Underflow). To fix this, we take the logarithm of the probabilities.
Instead of multiplying $0.01 \times 0.001$, we add their base-10 logarithms.
Calculate the sum of $\log_{10}(0.01)$ and $\log_{10}(0.001)$.

<details>
  <summary>Show solution</summary>
    
<ul>
  <li><strong>Step 1:</strong> Find the logarithm of each probability (what power of 10 equals the number?).<br>
  log<sub>10</sub>(0.01) = log<sub>10</sub>(10<sup>-2</sup>) = -2<br>
  log<sub>10</sub>(0.001) = log<sub>10</sub>(10<sup>-3</sup>) = -3</li>
  <li><strong>Step 2:</strong> Add the logarithms together (Log Rule: log(<i>A</i> &times; <i>B</i>) = log(<i>A</i>) + log(<i>B</i>)).<br>
  -2 + (-3) = -5</li>
  <li><strong>Answer:</strong> The sum is <strong>-5</strong>. By converting multiplication of tiny fractions into addition of negative integers, the computer easily handles the math without rounding to zero.</li>
</ul>

</details>
