

# Notation Reference

### L27: ML Foundations

### Logistic Regression

| Notation | Definition |
| --- | --- |
| $P(y=1 \mid x)$ | The probability that the target variable $y$ belongs to class 1 (e.g., an engine failure), given the input feature vector $x$. |
| $e$ | Euler's number ($\approx 2.718$), the base of the natural logarithm used in the sigmoid function to squash outputs between 0 and 1. |
| $\beta_0$ | The intercept (or bias) term of the linear equation. |
| $\beta_1, \dots, \beta_n$ | The weights (or coefficients) the model assigns to each respective feature. |
| $x_1, \dots, x_n$ | The individual data features (e.g., vibration, temperature) fed into the model. |
| $z$ | Used in the example to represent the raw linear sum (the weighted features plus the intercept) before it is passed into the sigmoid function. |

### Linear Discriminant Analysis (LDA)

| Notation | Definition |
| --- | --- |
| $P(Y=k \mid X=x)$ | The posterior probability that the observation $X$ belongs to a specific class $k$, given the evidence $x$. |
| $f_k(x)$ | The probability density function for class $k$. Represents how likely you are to see feature values $x$ assuming the observation belongs to class $k$. |
| $\pi_k$ | The prior probability of class $k$. The baseline historical frequency of the class before seeing any new evidence. |
| $K$ | The total number of possible classes. |
| $\sum_{l=1}^{K}$ | The sum of the probabilities across all possible classes, used as the denominator to normalize the final output into a valid percentage. |

### Quadratic Discriminant Analysis (QDA)

| Notation | Definition |
| --- | --- |
| $\delta_k(x)$ | The discriminant score for class $k$. The model calculates this score for all classes and classifies the observation as the class with the highest score. |
| $\Sigma_k$ | The covariance matrix for class $k$. Unlike LDA, QDA allows each class to have its own variance/spread. |
| $\lvert \Sigma_k \rvert$ | The determinant of the covariance matrix. |
| $\Sigma_k^{-1}$ | The inverse of the covariance matrix. |
| $\mu_k$ | The mean (average) vector of the features for class $k$. |
| $x^T$ | The transpose of the feature vector $x$ (used for matrix multiplication). |
| $\sigma_k^2$ | The variance of class $k$ (used in the simplified 1D formula). |
| $\ln()$ | The natural logarithm. |

### Support Vector Machines (SVM)

| Notation | Definition |
| --- | --- |
| $w$ | The weight vector. This defines the orientation of the hyperplane separating the classes. |
| $\lVert w \rVert$ | The magnitude (or Euclidean norm) of the weight vector. |
| $b$ | The bias term, which defines the offset of the hyperplane from the origin. |
| $\min_{w,b}$ | The mathematical objective function. Instructs the algorithm to find values of $w$ and $b$ that minimize $\frac{1}{2} \lVert w \rVert^2$, maximizing the margin between classes. |

### The Bias-Variance Tradeoff

| Notation | Definition |
| --- | --- |
| $E[(y - \hat{f}(x))^2]$ | The expected Mean Squared Error of the model. |
| $y$ | The true, actual target value in the real world. |
| $\hat{f}(x)$ | The predicted value generated by the model based on input $x$. |
| $\text{Bias}(\hat{f}(x))^2$ | The squared bias. Represents error introduced by the model making overly simplistic assumptions (underfitting). |
| $\text{Var}(\hat{f}(x))$ | The variance. Represents error introduced by the model being overly sensitive to noise in the training data (overfitting). |
| $\sigma^2$ | The irreducible error. Represents the inherent randomness or "fog of war" in the system that no model can ever predict. |

### Decision Trees & Gini Impurity

| Notation | Definition |
| --- | --- |
| $Gini$ | The Gini Impurity score. A measure of how "mixed" a node is. A score of $0$ means the node is perfectly pure (contains only one class). |
| $C$ | The total number of different classes present in that specific node. |
| $p_i$ | The probability (or proportion) of an observation belonging to class $i$ within that specific node. |
| $\sum_{i=1}^{C}$ | The sum of the squared probabilities for all classes in the node. |

## L28: Naive Bayes Classification

### The Core Concept: Bayes' Theorem

| Notation | Definition |
| --- | --- |
| $P(\text{Class} \mid \text{Evidence})$ | **The Posterior:** The updated probability that a data point belongs to a specific Class, given the new Evidence just observed. |
| $P(\text{Evidence} \mid \text{Class})$ | **The Likelihood:** The probability of observing this specific Evidence, assuming the data point *does* belong to that Class. |
| $P(\text{Class})$ | **The Prior:** The baseline historical probability of the Class occurring, before any new evidence is considered. |
| $P(\text{Evidence})$ | **The Marginal Probability:** The total probability of observing this evidence across all possible classes. |

### Text Classification

| Notation | Definition |
| --- | --- |
| $w_1, w_2, \dots, w_n$ | A sequence of individual words in a text document or message (e.g., $w_1 =$ "Enemy", $w_2 =$ "spotted"). |
| $\propto$ | "Proportional to." Indicates a relative score rather than a true probability, as the marginal probability denominator is dropped. |
| $\prod_{i=1}^{n}$ | The mathematical product operator. Instructs the algorithm to multiply a sequence of terms together. |

### Zero-Frequency Events & Laplace Smoothing

| Notation | Definition |
| --- | --- |
| $P(w \mid C)$ | The smoothed likelihood probability of word $w$ occurring in Class $C$. |
| $\text{count}(w, C)$ | The raw number of times word $w$ historically appeared in Class $C$ during training. |
| $\alpha$ | The smoothing parameter. Usually set to $1$ (Add-One smoothing) to mathematically pretend we have seen every word at least once. |
| $\lvert V \rvert$ | The total size of the vocabulary. Represents the number of *unique* words the model knows about across all classes combined. |

## L29: Neural Networks I

### The Artificial Neuron (The Perceptron)

| Notation | Definition |
| --- | --- |
| $z$ | The weighted sum (logit or pre-activation value). The raw combination of inputs, weights, and bias before the activation function. |
| $x_i$ | A single input feature (e.g., Payload Weight). |
| $w_i$ | The weight assigned to input $x_i$. Determines how important a specific input is to the final prediction. |
| $b$ | The bias term. A learnable constant that acts as a baseline threshold, allowing the neuron to "fire" even if all input features are zero. |
| $\sum_{i=1}^{n}$ | The summation operator. Instructs the algorithm to multiply every input by its corresponding weight and sum them up. |
| $e$ | Euler's number ($\approx 2.718$), the base of the natural logarithm used in the sigmoid function. |

### Feedforward Architectures & Non-Linearity

| Notation | Definition |
| --- | --- |
| $f(x)$ | A generalized function notation representing the output of a specific activation function given an input $x$. |
| $\max(0, x)$ | The mathematical definition of the **ReLU** (Rectified Linear Unit) function. It returns $x$ if positive, otherwise $0$. |
| $\sigma(x)$ | The standard symbol for the **Sigmoid** activation function, mapping any real number $x$ into a probability bounded between 0 and 1. |
| $X$ | A vector (array) representing all the input features for a single observation. |
| $z_{out}$ | The weighted sum calculated specifically at the final **Output Node** of the neural network, right before the final prediction is made. |


## L30: Neural Networks II

### Gradient Descent & Loss

| Notation | Definition |
| --- | --- |
| $L$ | **Loss (Cost):** The calculated error of the neural network's prediction. The network's entire goal is to minimize this number. |
| $\hat{y}$ | (Y-hat). The predicted output generated by the network. |
| $y$ | The true, actual target value the network was attempting to predict. |
| $\nabla L$ | The **Gradient**. A vector of partial derivatives representing the direction of steepest ascent for the Loss function. |
| $\frac{\partial L}{\partial w}$ | The partial derivative of the Loss with respect to a specific weight $w$. It tells the network how much that specific weight contributed to the total error. |
| $\alpha$ | **Learning Rate (Alpha):** A hyperparameter that controls the step size the network takes during weight updates. |
| $w_{new}$ | The updated weight after a single step of Gradient Descent. |

### Regularization

| Notation | Definition |
| --- | --- |
| $p$ | **Dropout Rate:** The probability (a fraction between 0 and 1) that any given neuron in a specific layer will be temporarily disabled during a single training step. |
