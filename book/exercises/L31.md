
## L31: Transfer learning, Autoencoders

**1. Transfer Learning Parameter Calculation**

You are adapting a massive visual model to classify drone damage. The pre-trained base model has $14,000,000$ weights. You freeze the base model. You add a GlobalAveragePooling layer (which has 0 weights), a hidden Dense layer with $100$ neurons, and an output layer with $4$ neurons. The input to the hidden Dense layer is a vector of $500$ features.
Calculate the total number of **trainable** parameters. (Remember, each neuron requires one weight per input, plus one bias).

<details>
  <summary>Show solution</summary>

**Step 1:** Calculate the parameters for the hidden layer. (500 inputs $\times$ 100 neurons) + 100 biases.

* $50,000 + 100 = 50,100$


**Step 2:** Calculate the parameters for the output layer. (100 inputs from the hidden layer $\times$ 4 neurons) + 4 biases.

* $400 + 4 = 404$


**Step 3:** Sum the trainable parameters.

* $50,100 + 404 = 50,504$


**Answer:** You only have to train **50,504** parameters, drastically reducing computation compared to the 14 million frozen parameters.

</details>

**2. Autoencoder Compression Ratio**

You are building an Autoencoder for a satellite communications array. The raw input data is a vector of $10,000$ telemetry features. Your Encoder consists of three layers: dropping from $10,000 \rightarrow 1,000 \rightarrow 250 \rightarrow 10$. The $10$-neuron layer is the latent space bottleneck. What is the mathematical compression ratio of this network?

<details>
  <summary>Show solution</summary>

**Step 1:** Identify the input size ($10,000$) and the bottleneck size ($10$).

**Step 2:** Divide the input size by the bottleneck size.
* $10,000 / 10 = 1,000$


**Answer:** The compression ratio is **1000:1**. The network is forced to represent the entire telemetry state using only $0.1\%$ of its original volume, guaranteeing it only learns the most critical underlying patterns.

</details>

**3. Reconstruction Error Thresholding**

Your deployed cyber Autoencoder has a baseline average reconstruction MSE of $0.4$ with a standard deviation of $0.1$. The commander mandates a strict 3-Sigma rule: any network packet causing a reconstruction error greater than 3 standard deviations above the mean must be flagged as anomalous. An incoming packet generates a reconstructed MSE of $0.75$. Does this packet trigger the alarm?


<details>
  <summary>Show solution</summary>

**Step 1:** Calculate the threshold. (Mean) + (3 $\times$ Standard Deviation).

* $0.4 + (3 \times 0.1) = 0.4 + 0.3 = 0.7$


**Step 2:** Compare the incoming packet's error to the threshold.

* $0.75 > 0.70$


**Answer:** **Yes**. The packet exceeds the threshold and triggers an automatic network quarantine.

</details>
