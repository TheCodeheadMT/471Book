**1. Calculating Mean Squared Error (MSE)**
You are evaluating a targeting network. Over a mini-batch of 3 training drops, you compare the network's predicted impact deviation (in meters) against the actual physical deviation.

<details>
  <summary>Show solution</summary>

* Drop 1: Actual = 10, Predicted = 8
* Drop 2: Actual = -5, Predicted = -2
* Drop 3: Actual = 2, Predicted = 2

Calculate the Mean Squared Error (Loss) for this mini-batch.

**Step 1:** Calculate the raw error (Actual - Predicted) for each drop.
- Drop 1: $10 - 8 = 2$
- Drop 2: $-5 - (-2) = -3$
- Drop 3: $2 - 2 = 0$

**Step 2:** Square each error.
* $2^2 = 4$
* $(-3)^2 = 9$
* $0^2 = 0$


**Step 3:** Calculate the mean (average) of the squared errors.
* $L = \frac{4 + 9 + 0}{3} = \frac{13}{3} \approx 4.33$


**Answer:** The Mean Squared Error for this batch is **4.33**.

**2. A Single Gradient Descent Step**
During backpropagation, the network evaluates a specific weight governing the aileron control of a drone. The current weight is $w_{old} = 1.5$. The calculated gradient for this weight is $2.0$ (meaning increasing the weight causes the error to go up). The network's Learning Rate is $\alpha = 0.1$. Calculate the new weight.

<details>
  <summary>Show solution</summary>

**Step 1:** Multiply the Learning Rate by the gradient to find the step size.
* $\text{Step} = 0.1 \times 2.0 = 0.2$


**Step 2:** Subtract the step from the old weight (moving in the opposite direction of the gradient).
* $w_{new} = 1.5 - 0.2 = 1.3$


**Answer:** The updated weight is **1.3**. The network successfully reduced the weight to lower the error.

</details>

**3. The Danger of a High Learning Rate (Overshooting)**
Using the exact same drone from Problem 2, imagine an inexperienced Data Scientist sets the Learning Rate far too aggressively: $\alpha = 1.5$. The starting weight is still $w_{old} = 1.5$, and the gradient is still $2.0$. Calculate the new weight and interpret the result.

<details>
  <summary>Show solution</summary>


**Step 1:** Calculate the step size.
* $\text{Step} = 1.5 \times 2.0 = 3.0$


**Step 2:** Subtract the step from the old weight.
* $w_{new} = 1.5 - 3.0 = -1.5$


**Answer:** The new weight violently swings to **-1.5**. Instead of carefully stepping down the gradient canyon, the massive learning rate caused the network to jump entirely across the canyon to the other side. This causes erratic training and prevents the network from ever finding the optimal target weight.

</details>

**4. Network Capacity and Dropout**
You are designing an Electronic Warfare classification network. A massive hidden layer contains 512 artificial neurons. To prevent the network from memorizing the specific background static of the training data, you implement a Dropout layer with a rate of $p = 0.375$. On average, how many neurons will remain *active* in this layer during a single training pass?

<details>
  <summary>Show solution</summary>

**Step 1:** Calculate the percentage of active neurons ($1 - p$).
* $1.0 - 0.375 = 0.625$ (or 62.5%)


**Step 2:** Multiply the total number of neurons by the active percentage.
* $\text{Active Neurons} = 512 \times 0.625 = 320$


**Answer:** Only **320** neurons will be active. By randomly silencing 192 neurons every pass, the network is forced to build redundant, generalized feature detectors.

</details>

**5. Diagnosing Overfitting from Training Logs**
You are reviewing the training logs of an experimental sensor fusion model over 100 epochs.

* Epoch 10: Training Loss = 1.50, Validation Loss = 1.60
* Epoch 50: Training Loss = 0.40, Validation Loss = 0.45
* Epoch 80: Training Loss = 0.10, Validation Loss = 0.90
* Epoch 100: Training Loss = 0.02, Validation Loss = 1.85
At approximately what epoch did the network begin to memorize the "fog of war" (overfit), and what technique should you use to capture the best version of this model?

<details>
  <summary>Show solution</summary>

**Step 1:** Compare the trend of Training Loss against Validation Loss. They drop together until Epoch 50.

**Step 2:** Observe the divergence. By Epoch 80, Training Loss continues to drop (0.10), but Validation Loss has spiked (0.90). This divergence indicates the network is failing on unseen data.

**Answer:** Overfitting began shortly after **Epoch 50**. You should implement an **Early Stopping** callback configured to monitor `val_loss` with a patience of ~10 epochs. This will automatically halt training and restore the optimal weights from the bottom of the validation curve (around Epoch 50) before the catastrophic divergence occurs.

</details>

**6. A Complete Forward and Backward Pass (Linear Network)**
Let's manually execute a single network cycle to see how prediction, error, and learning connect.
A highly simplified network with linear activation receives an input $x = 2$. The target true value is $y = 10$. The initial weight is $w = 3$ (assume bias is 0). The Learning Rate is $\alpha = 0.05$.

Given that the gradient formula for an MSE loss of a single sample without bias is $\nabla = -2x(y - wx)$, calculate the updated weight.

<details>
  <summary>Show solution</summary>

**Step 1 (Forward Pass):** Calculate the prediction ($\hat{y} = w \times x$).
* $\hat{y} = 3 \times 2 = 6$


**Step 2 (Loss Calculation):** The true target is 10, but the network predicted 6.
* $\text{Error} = 10 - 6 = 4$


**Step 3 (Calculate Gradient):** Use the provided derivative formula.
* $\nabla = -2(2) \times (10 - 6) = -4 \times 4 = -16$


**Step 4 (Gradient Descent Update):** Apply the weight update rule $w_{new} = w_{old} - \alpha \times \nabla$.
* $w_{new} = 3 - (0.05 \times -16)$
* $w_{new} = 3 - (-0.8) = 3.8$


**Answer:** The new weight is **3.8**.
*(Verification: If we run a second forward pass with the new weight, $\hat{y} = 3.8 \times 2 = 7.6$. The prediction moved closer to the true target of 10! The network is successfully learning.)*

</details>
