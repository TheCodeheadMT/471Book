## L32 CNN, Max Pooling


**1. Output Dimension Calculation (Valid Padding)**
You are designing a CNN to process 256x256 pixel imagery from a targeting pod ($W = 256$). You apply a convolution layer with a 5x5 filter ($F = 5$), a stride of 1 ($S = 1$), and no padding ($P = 0$). Calculate the spatial dimensions (width and height) of the resulting feature map.

<details>
  <summary>Show solution</summary>

**Step 1:** Use the spatial formula: 

* $O = \lfloor \frac{W - F + 2P}{S} \rfloor + 1$.

**Step 2:** Substitute the values: 

* $O = \lfloor \frac{256 - 5 + 0}{1} \rfloor + 1$.

**Step 3:** Calculate:

* $O = 251 + 1 = 252$.

**Answer:** The output feature map will be **252x252**. Because we did not use padding, we lost 2 pixels on every edge.


</details>

**2. Output Dimension Calculation (Strided)**
To aggressively reduce computational load, you take the 252x252 output from Problem 1 ($W = 252$) and apply a new Convolutional layer with a 3x3 filter ($F = 3$), but this time you use a stride of 3 ($S = 3$) and no padding ($P = 0$). Calculate the new output dimension.

<details>
  <summary>Show solution</summary>

**Step 1:** Substitute the values:

* $O = \lfloor \frac{252 - 3 + 0}{3} \rfloor + 1$.

**Step 2:** Solve the division: 

* $\frac{249}{3} = 83$.

**Step 3:** Add 1:

* $83 + 1 = 84$.

**Answer:** The output shrinks massively to **84x84**.



</details>

**3. Max Pooling Operation**
A radar processing network outputs the following 4x4 feature map:

```text
[ 1,  3,  2,  4 ]
[ 5,  8,  1,  0 ]
[ 2,  0,  6,  7 ]
[ 4,  1,  3,  9 ]

```

Apply a 2x2 Max Pooling window with a stride of 2 ($S = 2$). What does the resulting 2x2 matrix look like?

<details>
  <summary>Show solution</summary>

**Step 1:** Look at the top-left 2x2 quadrant: 

* `[1, 3], [5, 8]`. The max value is 8.

**Step 2:** Shift the window right by 2 (top-right quadrant): 

* `[2, 4], [1, 0]`. The max value is 4.

**Step 3:** Shift the window down to the bottom-left quadrant: 

* `[2, 0], [4, 1]`. The max value is 4.

**Step 4:** Look at the bottom-right quadrant: 

* `[6, 7], [3, 9]`. The max value is 9.

**Answer:** The resulting pooled matrix is:

```text
[ 8,  4 ]
[ 4,  9 ]

```

**4. Parameter Calculation in CNNs vs Dense Networks**
Imagine processing a 100x100 pixel greyscale image.
If you flatten this into a 10,000-element 1D vector and feed it into a standard Dense layer with 10 neurons, how many weights must be learned?
Conversely, if you use a Convolutional layer with 10 distinct 3x3 filters, how many weights must be learned?

<details>
  <summary>Show solution</summary>

**Step 1 (Dense):** 

* 10,000 inputs $\times$ 10 neurons = **100,000 weights** (plus 10 biases).

**Step 2 (CNN):** A 3x3 filter has 9 weights. You have 10 filters. 

* 9 $\times$ 10 = **90 weights** (plus 10 biases).

**Answer:** The CNN only needs to learn **90 weights**, while the dense network must learn **100,000**. CNNs are vastly more efficient because a single filter slides across the entire image, reusing the exact same weights (parameter sharing).
</details>
